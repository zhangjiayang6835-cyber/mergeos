# Code of Conduct

## Our Pledge

MergeOS is a public bounty and software maintenance project. We want this repository to be a respectful, practical, and safe place for contributors, maintainers, customers, and reviewers.

Everyone who participates in issues, pull requests, reviews, discussions, or bounty claims is expected to follow this code of conduct.

## Expected Behavior

- Be respectful, patient, and constructive.
- Keep feedback focused on the work, evidence, tests, and user impact.
- Assume good intent, but accept correction when something causes harm.
- Give clear context when reporting bugs, claiming bounties, or requesting review.
- Protect private data, secrets, customer information, and production details.
- Follow the bounty claim and evidence process before asking for payout review.

## Unacceptable Behavior

- Harassment, threats, insults, personal attacks, or discriminatory language.
- Spam, low-effort bounty claims, duplicate PR flooding, or intentionally misleading evidence.
- Publishing secrets, tokens, credentials, private keys, customer data, or private infrastructure details.
- Social engineering, doxxing, or attempts to bypass the security policy.
- Repeatedly ignoring maintainer guidance after a clear warning.

## Reporting

For conduct issues, contact the maintainers through the repository owner or a private GitHub message when available. Do not open a public issue for private personal safety concerns.

For security vulnerabilities, follow [SECURITY.md](SECURITY.md) instead of posting public details.

## Enforcement

Maintainers may remove comments, close issues or PRs, reject bounty claims, block users, or report abuse to GitHub when behavior violates this policy.

Enforcement decisions are based on impact, severity, pattern of behavior, and risk to the project or its users.
